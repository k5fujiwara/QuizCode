# 技術ポイント

このクイズアプリは、HTML / CSS / JavaScript / JSON だけで作られた静的Webアプリです。

サーバーやデータベースを使わずに動くため、ダウンロードまたはクローンしてすぐに中身を確認できます。GitHub Pages にそのまま公開できるので、Webアプリの基本を学ぶ教材や、プログラミング学習者向けの改造用テンプレートとして使いやすい構成です。

GitHub:

```text
https://github.com/k5fujiwara/QuizCode
```

## このテンプレートで学べること

- HTMLで画面の構造を作る方法
- CSSで見た目を整える方法
- JavaScriptで画面を動かす方法
- JSONでデータを管理する方法
- `fetch()` で外部JSONを読み込む方法
- DOM操作で要素を作成・更新する方法
- ボタンクリックなどのイベント処理
- クイズの採点ロジック
- ランダム出題や選択肢シャッフル
- タイマー処理
- GitHub Pagesで静的サイトを公開する方法

## アプリの全体像

このアプリは、大きく分けると3つの層でできています。

```text
HTML     画面の土台
CSS      見た目・レイアウト
JavaScript  動き・読み込み・採点
JSON     クイズ問題・単元情報
```

一般的なWebアプリでは、問題データをサーバーやデータベースから取得することがあります。このテンプレートでは、学習しやすさを優先して、問題データを JSON ファイルとして管理しています。

そのため、初心者でも「問題を増やす」「文言を変える」「見た目を変える」といった改造から始めやすくなっています。

## 静的Webアプリとして動く

このアプリは、サーバー側のプログラムを必要としません。

必要なファイルは主に以下です。

```text
index.html
styles.css
app.js
data/catalog.json
sample-unit.json
```

`index.html` を入口として、CSSとJavaScriptを読み込みます。JavaScriptは `data/catalog.json` と問題JSONを読み込み、画面にクイズを表示します。

静的ファイルだけで構成されているため、GitHub Pages で公開できます。

## JSONで問題データを管理する

問題データは `sample-unit.json` のようなJSONファイルで管理します。

例:

```json
{
  "unit_title": "サンプル単元（Web・プログラミング基礎）",
  "questions": [
    {
      "question": "HTMLでページの見出しを表す要素として最も適切なものはどれですか？",
      "options": ["<header>", "<h1>", "<title>", "<section>"],
      "answer": 1,
      "commentary": "見出しの主役は h1〜h6 です。title は文書タイトル、header は領域のヘッダーです。"
    }
  ]
}
```

この形式にすることで、プログラム本体を触らなくても問題を追加・編集できます。

学習ポイント:

- 配列とオブジェクトの構造
- 文字列、数値、配列の扱い
- JSONの構文ルール
- データと画面表示の分離

## catalog.jsonでコース・単元を管理する

`data/catalog.json` は、コース・分野・単元の一覧を管理するファイルです。

アプリは最初にこのファイルを読み込み、どの単元JSONを選べるようにするかを決めます。

```json
{
  "version": 2,
  "courses": [
    {
      "id": "sample",
      "name": "サンプルコース",
      "fields": [
        {
          "id": "sample-basic",
          "name": "サンプル分野",
          "units": [
            {
              "id": "sample-u1",
              "title": "サンプル単元",
              "jsonPath": "sample-unit.json"
            }
          ]
        }
      ]
    }
  ]
}
```

このように一覧データを別ファイルにすることで、アプリ本体を変更せずにコースや単元を増やせます。

学習ポイント:

- 設定ファイルの考え方
- データ駆動のUI
- ファイルパスの扱い
- 拡張しやすいデータ構造

## fetchでJSONを読み込む

JavaScriptでは `fetch()` を使ってJSONを読み込んでいます。

この仕組みにより、HTMLの中に問題を直接書かず、外部ファイルからデータを読み込む構成になっています。

学習ポイント:

- 非同期処理
- `async` / `await`
- HTTP経由でファイルを読み込む仕組み
- 読み込み失敗時のエラー処理

注意点として、ブラウザで `index.html` を直接開くと、環境によってはJSONを読み込めません。そのため、ローカルでは Live Server や `python -m http.server` などの簡易サーバーを使います。

## DOM操作で画面を作る

問題文や選択肢は、JavaScriptでHTML要素を作って表示しています。

たとえば、選択肢ボタンは問題データに応じて動的に生成されます。

学習ポイント:

- `document.createElement()`
- `textContent`
- `innerHTML`
- `appendChild()`
- `classList`
- `hidden` 属性による表示切り替え

HTMLにすべての問題を書いておくのではなく、データに合わせて画面を作ることで、問題数が増えても同じ仕組みで表示できます。

## イベント処理

ボタンをクリックしたときの処理には、`addEventListener()` を使っています。

例:

- コースを選んだときに単元一覧を更新する
- 単元を読み込む
- クイズモードを開始する
- チャレンジモードを開始する
- 選択肢をクリックして回答する
- 提出ボタンで採点する

学習ポイント:

- クリックイベント
- フォーム操作
- 選択状態の管理
- ユーザー操作に応じた画面更新

## クイズモード

クイズモードでは、1問ずつ問題を表示します。

ユーザーが選択肢をクリックすると、すぐに正誤判定と解説が表示されます。学習用として、間違えた直後に解説を読めるのが特徴です。

主な処理:

- 現在の問題番号を管理する
- 選択肢を表示する
- 正解・不正解を判定する
- 解説を表示する
- 次の問題へ進む
- 最後に結果を表示する

学習ポイント:

- 状態管理
- 条件分岐
- スコア計算
- 進捗表示

## チャレンジモード

チャレンジモードでは、複数の問題をまとめて解き、最後に採点します。

制限時間があり、時間切れになると自動提出されます。小さなアプリの中で、テスト形式のUIやタイマー処理を学べます。

主な処理:

- 出題する問題をランダムに選ぶ
- 選択肢の順番をシャッフルする
- 回答済みの数を表示する
- 残り時間を表示する
- 提出時にまとめて採点する
- 合格・不合格を表示する

学習ポイント:

- 配列操作
- ランダム処理
- `setInterval()`
- タイマーの開始・停止
- 自動提出処理
- 採点結果の一覧表示

## ランダム出題と選択肢シャッフル

チャレンジモードでは、問題や選択肢をランダムに並び替えます。

そのままシャッフルすると正解番号がずれてしまうため、選択肢を並び替えた後に「表示上の正解番号」を再計算しています。

学習ポイント:

- 配列のコピー
- Fisher-Yatesシャッフル
- インデックスの扱い
- データ変換

クイズアプリでは、単にランダムにするだけでなく、正解との対応関係を保つことが重要です。

## 入力データのチェック

読み込んだJSONは、アプリ内で最低限の形式チェックをしています。

チェック例:

- `questions` が配列になっているか
- 選択肢が4件あるか
- `answer` が `0` から `3` の整数か
- 単元タイトルがあるか

学習ポイント:

- バリデーション
- エラーメッセージ
- 想定外データへの対応

教材テンプレートでは、JSONの書き間違いが起こりやすいため、読み込み時のチェックが重要です。

## HTMLを安全に表示する工夫

問題文や解説では、コード例を表示できるように一部のHTMLタグを許可しています。

一方で、どんなHTMLでもそのまま表示すると危険な場合があります。そのため、許可するタグを限定し、不要な属性は削除する処理を入れています。

学習ポイント:

- `innerHTML` の注意点
- 許可リスト方式
- 簡易的なサニタイズ
- セキュリティを意識した表示

このアプリは本格的なセキュリティ教材ではありませんが、「外部データを画面に出すときは注意が必要」という考え方を学べます。

## アクセシビリティへの配慮

例:

- `role="progressbar"`
- `role="timer"`
- `aria-hidden`

学習ポイント:

- スクリーンリーダーへの配慮

小さなアプリでも、アクセシビリティを意識した実装を学べます。

## 学習ステップ例

初心者がこのコードを使って学ぶ場合は、次の順番がおすすめです。

1. `sample-unit.json` の問題文を変える
2. 選択肢と正解番号を変える
3. 問題を追加する
4. `data/catalog.json` に単元を追加する
5. `index.html` のタイトルを変える
6. `styles.css` で色を変える
7. `app.js` の制限時間や出題数を変える
8. クイズモードの表示文言を変える
9. チャレンジモードの採点条件を変える
10. GitHub Pagesで公開する

## まとめ

このクイズアプリは、単なる問題表示アプリではなく、Webアプリ開発の基本要素を小さくまとめたテンプレートです。

JSONを編集するだけでも使えますが、中身を読んでいくと、HTML / CSS / JavaScript の役割、データ管理、イベント処理、DOM操作、タイマー、公開までの流れを一通り学べます。

「まず動くものを触る」「少し変えてみる」「公開して共有する」という流れを体験しやすい教材として活用できます。
